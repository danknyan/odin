Executive Summary:¶

Background:¶

Intended for content creators (and their fans), the ability to get understanding of the reaction to a video is mainly through its comments. Unfortunately, the YouTube channel interface does not provide any form of analytics on comments - at present. Additionally, instructional videos and the like depend on accuracy and/or replicability, through which the comments serve as a resource for content improvement and further viewer engagement. Thus, the content creator stands much to gain from the ability to quickly assess the discourse around a video, the negative/positive sentiment of comments, and commonly mentioned topics within.

Methodology:¶

The pipeline consists of Youtube API for comments, NLTK for corpus processing, HuggingFace for Sentiment Analysis, YouTubeAPI for transcripts and Matplotlib for results display. It accepts a url or video tag and outputs a graph displaying top # words, top # comments, and the first 140 characters of example comments that contain the first # of the top # words - as well as whether to filter top words/comments by whether or not they contain words used in the video itself. The addition of this transcript-based filter increases the relevancy of the output by selecting only words/comments that directly relate to the video's content.

Known Limitations:¶

  • Model Modality: The model has a single mode: sentiment analysis; it cannot provide in-depth information about the video itself or commonalities between comments beyond word sentiment and frequency. Future improvements may include a summary of all comments of a given sentiment or a summary of the video's content.
  • Model Accuracy: Top comments and word predictions are limited by the accuracy of the model. Comments containing sarcasm or humor and those with a narrative structure tend to mislead the predictions.
  • English Language Only: The model has been trained only using English and cannot effectively analyze other languages, though emojis do provide some guidance. In addition, Matplotlib cannot properly display 2-bit characters with default settings.
  • Transcript Filter: By filtering comments, it is possible that ommissions made during content creation that would have been valuable to results in comments regarding these ommission being overlooked. For example, a travel guide video that didn't visit a certain spot or viewer requests for the next destination. It is likely that the ideal results contain a mix of filtered and non-filtered content. Parenthetically, adding a filter for common English words might improve result relevance.
  • High Freqency Tokens: Naturally high-frequency, non-stopword tokens distort the output as well, including: "like","see", etc. These could be mitigated by applying an additional filter.

Conclusion:¶

This tool, both with and without content filters, can serve to help a content creator get a quick read on the reaction to their video and plan future content, despite sentiment accuracy-related shortcomings.

def get_comments(video_tag,verbose=False,api_key=api_key) -> tuple[pd.DataFrame, str]:
    youtube = build('youtube', 'v3', developerKey=api_key)

    # get title if needed
    video_request=youtube.videos().list( 
        part='snippet', 
        id=video_tag
        ) 
    video_response = video_request.execute()
    vid_title = video_response['items'][0]['snippet']['title']

    # get comments
    request = youtube.commentThreads().list(
        part="snippet,replies",
        videoId=video_tag,
        textFormat="plainText"
    )
    # init empty df
    df = pd.DataFrame(columns=['comment','likes'])


    while request:
        comments = []
        likes = []
        try:
            response = request.execute()
            #counter for verbosity
            cycle_count=0
            
            for item in response['items']:
                # Extracting comments
                comment = item['snippet']['topLevelComment']['snippet']['textDisplay']
                comments.append(comment)
                # Extracting likes
                like_count = item['snippet']['topLevelComment']['snippet']['likeCount']
                likes.append(like_count)

            # create new dataframe
            df2 = pd.DataFrame.from_dict({"comment": comments, "likes":likes},orient="index").transpose()
            # update existing df 
            df = pd.concat([df, df2], ignore_index=True)
            # save df just in case
            df.to_json(f"./data/user_comments_{video_tag}.json")
            #get next page
            request = youtube.commentThreads().list_next(request, response)
            #report progress
            if (verbose == True) and (cycle_count % 50 == 0):
                print(f"Iterating. {df.shape[0]} comments detected thus far.")
            cycle_count +=1
        except Exception as e:
            print(f"Reached suspected end-of-page. Exporting to {video_tag}_user_comments.json .")
            df.to_json(f"./data/user_comments_{video_tag}.json")
            print(str(e))
            print(traceback.format_exc())
    return df,vid_title

# Preprocess text (username and link placeholders)
def preprocess(text:str,remove_nonlexical=False,return_list=True,model_size=514):
    # skip empty / http ->spam
    if (not text) or ("https" in text):
        return ""

    #remove newline and split to list
    if isinstance(text,str):
        # skip blanks and non-Latin
        if not (text[-1].isascii() or text[0].isascii()):
            return ""
        text = text.lower()
        text = text.replace("\n"," ") # remove new line
        # tokenize or just split
        if remove_nonlexical:
            text_as_list = word_tokenize(text)
        else:
            text_as_list = text.split(" ")
    #empty list
    new_text = []
    # for each word in a text
    for t in text_as_list:
        # skip if blank or non-latin
        if (not t or not t[0].isascii()):
            continue
        # anonymization
        t = '@user' if t.startswith('@') and len(t) > 1 else t
        t = 'http' if t.startswith('http') else t

        # lexical processing
        if remove_nonlexical:
            t = remove_punctuation(t) # remove punctuation
            #remove stopwords and blanks
            if (not t or (t in stop_words or t in "nt") or t[-1].isnumeric()):
                continue
        new_text.append(t.lower())
    #limit by model buffer size
    if return_list:
        return " ".join(new_text)[:model_size]
    return new_text

def process_comment_from_df(raw_text:str) -> tuple[float,float,float]:
    # preprocess text
    text = preprocess(raw_text)
    encoded_input = tokenizer(text, return_tensors='pt')
    # model output
    try:
        output = model(**encoded_input)
        scores = softmax(output[0][0].detach().numpy())
    
    except Exception as e:
        print("raw text:",raw_text)
        print("processed text:",text)
        print(str(e))
        #print(traceback.format_exc())
        scores = np.zeros(3)
 
    return list(scores)

def sentiment_analysis(df:pd.DataFrame) -> pd.DataFrame:
    senti = df["comment"].map(lambda x: process_comment_from_df(x))
    negative = list(map(lambda x: x[0],senti))
    neutral = list(map(lambda x: x[1],senti))
    positive = list(map(lambda x: x[2],senti))
    senti_df = pd.DataFrame.from_dict({"negative":negative,"neutral":neutral,"positive":positive})
    return pd.concat([senti_df,df],axis=1)

def build_dictionary(df:pd.DataFrame,return_ngrams=False) -> Counter:
    unigrams_freq = Counter()
    bigrams_freq = Counter()
    bigrams = []
    # TODO: fix bigram generator
    # get tokens
    tokens = df["comment"].map(lambda x: word_tokenize(preprocess(x,remove_nonlexical=True)))
    # get ngrams
    for comment in tokens:
        comment_bigrams = []
        for i in range(len(comment)-1):
            comment_bigrams.append(" ".join(comment[i:i+2]))
        bigrams.append(comment_bigrams)
    # update dictionary
    list(map(lambda x:unigrams_freq.update(x),tokens))
    # update bigrams counter
    list(map(lambda x:bigrams_freq.update(x),bigrams))

    # add columns
    df["unigrams"]=tokens
    df["bigrams"]=bigrams
    if return_ngrams:
        return unigrams_freq,bigrams_freq

# get ngram sentiment
def get_ngram_sentiment(df) -> dict:
    # setup vars
    pos_tokens = Counter()
    neg_tokens = Counter()
    for _,row in df.iterrows():
        # check sentiment, make entry
        if (row["negative"] > row["positive"]):
            neg_tokens.update(row["unigrams"])
            neg_tokens.update(row["bigrams"])
        else:
            pos_tokens.update(row["unigrams"])
            pos_tokens.update(row["bigrams"])

    return pos_tokens,neg_tokens

# return only top words that are more prevealent in dict_a than dict_b
def curate_dicts(word_counter_a:Counter,word_counter_b:Counter,transcript_filter=False):
    result_dict = Counter()
    for word, freq in word_counter_a.items():
        if transcript_filter and word not in transcript_filter:
            continue
        if (word not in eng_most_freq) and (word_counter_b.get(word,0) < freq) and (len(word.replace(" ",""))>2):
            result_dict[word] = freq
    return result_dict.most_common()

def find_examples_vectorized(df:pd.DataFrame, top_words:list[str],sentiment:str, max_length:int):
    top_com = []
    if sentiment=="Positive":
        df_comments = df[df["positive"]>df["negative"]].sort_values("likes",ascending=False)['comment']
    else:
        df_comments = df[(df["positive"]<df["negative"]) & ((df["neutral"]<df["negative"]))].sort_values("likes",ascending=False)['comment']
    truncated_comments = df_comments.str[:max_length]
    
    for word, _ in top_words[:5]:
        # Find matches using pandas string operations
        matches = truncated_comments[
            truncated_comments.str.contains(word, na=False) & 
            ~truncated_comments.isin(top_com)
        ]
        if not matches.empty:
            top_com.append(matches.iloc[0])
    return top_com

def top_list_to_display(df,pos_top,neg_top,max_words=10, max_comments=5,max_com_length=140):
    fig_lower_margin = 11+max_words*.1+max_comments*.1+max_com_length*0.01
    top_list = {}
    top_list["pos_list"] = top_list_to_str(pos_top,f"Positive Words ({len(pos_top)})",max_words)
    top_list["neg_list"] = top_list_to_str(neg_top,f"Negative Words ({len(neg_top)})",max_words)
    pos_com=find_examples_vectorized(df,pos_top,"Positive",max_com_length)
    top_list["pos_com_list"] = top_list_to_str(pos_com,f"Positive Comments",max_comments)
    neg_com=find_examples_vectorized(df,neg_top, "Negative",max_com_length)
    top_list["neg_com_list"] = top_list_to_str(neg_com,f"Negative Comments",max_comments)
    return fig_lower_margin,top_list

# final pipeline for video analysis and visualization
def pipeline(video_tag:str,filter_by_transcript=False,max_words:int=10,max_comments:int=5,max_com_length=140,tlang="en"):
    if video_tag.startswith("http"):
        if "?v=" in video_tag:
            video_tag = video_tag.split("?v=")[1][:11]
        else:
            video_tag = video_tag.split("be/")[1][:11]
    df, vid_title = load_video_comments(video_tag)
    df_w_senti = sentiment_from_df(df,video_tag)
    if filter_by_transcript:
        transcript_set = get_set_from_transcript(video_tag,tlang)
    else:
        transcript_set=set()
    # get top words
    pos_top,neg_top = get_top_from_df(df_w_senti,transcript_set)
    # format tops words for display
    fig_height, top_list = top_list_to_display(df_w_senti,pos_top,neg_top,max_words, max_comments,max_com_length)
    visualize_sentiments(df_w_senti,vid_title,fig_height,top_list,video_tag)
